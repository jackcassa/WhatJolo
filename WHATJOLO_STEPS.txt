WhatJolo - stato attuale
========================

1. Progetto standalone
   - repository dedicato WhatJolo
   - solution dedicata WhatJolo.slnx
   - applicazione WPF .NET 8

2. Backend database
   - backend unico PostgreSQL
   - nessun fallback SQLite
   - nessuna dipendenza operativa da file DB locale
   - tab "Istanza DB" come prima tab
   - connessione esplicita tramite bottone "Connetti"
   - altre tab disabilitate finche' il DB non e' connesso

3. Configurazione istanza DB
   - host/porta/database/utente/password salvabili
   - test connessione dedicato
   - stato connessione mostrato in UI
   - inizializzazione con dati locali della macchina corrente

4. Struttura progetto
   - gestione progetti multipli
   - classi YOLO attive per progetto
   - metadata progetto nel DB:
     - ProjectRootPath
     - MachineName

5. Acquisizione ADB
   - start-server
   - refresh device
   - screencap PNG lossless
   - finestra preview floating dedicata
   - selezione crop fatta dalla finestra floating

6. Crop e annotazioni
   - salvataggio crop per classe
   - registrazione nel DB del progetto corrente
   - supporto variazioni crop
   - flag IsVariation per distinguere le variazioni
   - vista separata per crop normali e variazioni
   - cancellazione con tasto Canc

7. Allineamento progetto <-> DB
   - ogni acquisizione/crop viene allineata subito nel DB del progetto aperto
   - ogni immagine del progetto viene salvata anche come blob compresso
   - il progetto puo' essere ricreato dal DB sul file system
   - bottone "Apri/Ricrea progetto" con messaggio "Progetto caricato"

8. Dataset YOLO
   - creazione dataset train/val dal progetto selezionato
   - generazione anche dello split test a partire dai dati nel DB
   - immagini positive e negative incluse
   - labels YOLO generate automaticamente
   - image_map.tsv per il mapping immagine sorgente

9. Training YOLO
   - scelta modello YOLO
   - scelta profilo YOLO
   - monitor training con metriche percentuali:
     - Precision
     - Recall
     - mAP50
     - mAP50-95
   - commento qualita' leggibile
   - export ONNX
   - reset training

10. Test/Ultra
    - tab Ultra per test immagini
    - liste test/train/val
    - detection automatica sull'immagine selezionata
    - preview floating manuale
    - delete con Canc su test/train/val
    - rimozione anche delle selezioni collegate nel DB quando presenti

11. Vista Database
    - tab Database con stile black/yellow
    - elenco tabelle
    - filtro per progetto dove applicabile
    - refresh
    - delete singolo o multiplo con Canc

12. Stato build
    - build pulita verificata
    - 0 errori
    - 0 avvisi
